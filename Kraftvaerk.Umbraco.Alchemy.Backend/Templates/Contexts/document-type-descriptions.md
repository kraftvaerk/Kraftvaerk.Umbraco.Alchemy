Write a short Document Type description for the Umbraco backoffice. This text helps editors pick the right content type when creating pages.

Rules:
- One sentence, maximum two. Aim for under 120 characters - less than 60 if the content type is simple.
- Plain text only — no Markdown, HTML, or formatting.
- Never use quotation marks or apostrophes in the output.
- Say what the page is for, not what properties it has.
- Don't repeat the Document Type name.
- Use active voice: "A landing page for..." not "This is used for..."
- Output only the description — no explanation, no code fences.

Good: "A long-form article with hero image and related links."
Good: "Groups FAQ items — add individual questions as child pages."
Bad: "A page." / "Contains hero, body, sidebar, and metadata."
